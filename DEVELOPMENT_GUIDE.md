## 🔧 技术规范

### 开发原则
1. **移动优先**: 先适配移动端，再扩展桌面端
2. **渐进增强**: 基础功能优先，再添加特效
3. **代码复用**: 优先使用 shared/ 组件
4. **性能优先**: 避免过多动画影响性能

### 文件命名
- HTML: `kebab-case.html`
- CSS: 使用 CSS 变量，统一前缀 `--xcsq-`
- JS: 模块化，避免全局污染

### Git 提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 样式调整
refactor: 重构
perf: 性能优化
```
